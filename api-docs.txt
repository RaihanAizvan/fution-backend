Admin CMS API Reference

System is now pure Markdown-driven with cached HTML rendering. Content is stored as raw Markdown and pre-rendered to HTML on write (create/update) for high-performance delivery.

Base Path: /admin
Auth: Admin-only (guarded)
Content-Type: application/json

Subjects

GET /admin/subjects
List all subjects.

Response 200
[
  {
    "id": "uuid",
    "slug": "javascript",
    "title": "JavaScript",
    "description": "Optional",
    "orderIndex": 1,
    "isActive": true
  }
]

POST /admin/subjects
Create a subject.

Body
{
  "slug": "javascript",
  "title": "JavaScript",
  "description": "Optional",
  "orderIndex": 1,
  "isActive": true
}

Notes
- orderIndex auto-calculates if omitted.
- Slug must be unique.

PATCH /admin/subjects/:subjectId
Update a subject.

Body
{
  "slug": "javascript",
  "title": "JavaScript",
  "description": "Optional",
  "orderIndex": 1,
  "isActive": true
}

DELETE /admin/subjects/:subjectId
Delete a subject.

Rules
- Fails if any topic under the subject has a published version.
- If allowed, deletes subject, topics, and versions in a transaction.

Error
{
  "statusCode": 400,
  "message": "Cannot delete subject with published topic content. Deactivate it instead or unpublish the content first."
}

Response
{ "deleted": true }

PUT /admin/subjects/reorder
Bulk reorder subjects.

Body
{
  "subjects": [
    { "subjectId": "uuid-1", "orderIndex": 1 },
    { "subjectId": "uuid-2", "orderIndex": 2 }
  ]
}

POST /admin/subjects/import
Bulk import subject tree (atomic).

Body
{
  "subject": {
    "slug": "javascript",
    "title": "JavaScript",
    "orderIndex": 1,
    "isActive": true
  },
  "topics": [
    {
      "slug": "functions",
      "title": "Functions",
      "level": "beginner",
      "orderIndex": 1,
      "isActive": true,
      "versions": [
        {
          "status": "published",
          "markdown": "# Functions\n\nReusable blocks of logic.\n\n```javascript\nfunction add(a, b) { return a + b; }\n```"
        }
      ]
    }
  ]
}

Rules
- Subject is upserted by slug.
- Topics are upserted by slug + subjectId.
- Versions are always new (append-only).
- status: "published" unpublishes previous version, publishes the new one.
- HTML is automatically rendered from markdown during import.
- Entire import is transactional.

Validation Error
{
  "error": "VALIDATION_ERROR",
  "errors": [
    "topics[0].versions must not be empty",
    "topics[1].versions[0].markdown is required"
  ]
}

Topics

GET /admin/subjects/:subjectId/topics
List topics for a subject.

POST /admin/subjects/:subjectId/topics
Create a topic.

Body
{
  "slug": "functions",
  "title": "Functions",
  "level": "beginner",
  "orderIndex": 1,
  "isActive": true
}

Notes
- level is required.
- orderIndex auto-calculates if omitted.

PATCH /admin/subjects/:subjectId/topics/:topicId
Update a topic.

DELETE /admin/subjects/:subjectId/topics/:topicId
Delete a topic.

Rules
- Fails if the topic has any published version.
- If allowed, deletes topic and its versions in a transaction.

Response
{ "deleted": true }

PUT /admin/subjects/:subjectId/topics/reorder
Bulk reorder topics inside a subject.

Body
{
  "topics": [
    { "topicId": "uuid-1", "orderIndex": 1 },
    { "topicId": "uuid-2", "orderIndex": 2 }
  ]
}

Topic Versions

GET /admin/topics/:topicId/versions
List all versions for a topic.

GET /admin/topics/:topicId/versions/:versionId
Get a specific version (includes markdown and pre-rendered html).

POST /admin/topics/:topicId/versions
Create a draft version.

Body
{
  "version": 2,
  "markdown": "# Topic Content"
}

PATCH /admin/topics/:topicId/versions/:versionId
Update a version's content.

Body
{
  "markdown": "# Updated Content"
}

POST /admin/topics/:topicId/versions/:versionId/publish
Publish a version (unpublishes prior published version).

Public Content

GET /topics/:slug/content
Fetch published content for a topic (returns cached HTML).

Response 200
{
  "topic": { "slug": "string", "title": "string", "isActive": "boolean" },
  "markdown": "string",
  "html": "string (rendered HTML)"
}

Validation & Errors

Validation Pipe Enabled
- Requests are validated against DTOs.
- Unknown fields are rejected.

Common Error
{
  "statusCode": 400,
  "message": ["field must be a string"],
  "error": "Bad Request"
}
